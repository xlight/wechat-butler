# WeChat Butler 项目规划总结

## 项目概述

**WeChat Butler**（微信管家）是一个运行在个人电脑或 mini 设备上的超轻量微信消息自动化工具。它作为微信生态系统的"智能中枢"，连接现有的三个核心工具，形成一个完整的自动化解决方案。

### 项目定位
- **角色**: 微信消息自动化处理中心
- **目标用户**: 个人用户、小型团队
- **部署环境**: 个人电脑、树莓派等 mini 设备
- **核心价值**: 自动化、智能化、轻量化

## 核心需求分析

### 1. 消息自动化处理
- 接收 chatshell-api 的 webhook 通知
- 自动处理新消息，减少人工干预
- 支持多种消息类型（文本、图片、文件等）

### 2. 智能规则引擎
- 基于关键词、正则表达式等条件匹配
- 支持复杂的条件组合（AND/OR/NOT）
- 可配置的优先级和触发逻辑

### 3. 多样化动作支持
- 自动回复微信消息
- 消息转发和分发
- 执行系统命令和脚本
- 调用外部 API 接口
- 发送系统通知

### 4. AI 增强能力
- 集成 LLM 进行智能决策
- 自然语言理解和回复生成
- 上下文感知的对话管理

### 5. 外部控制接口
- HTTP RESTful API 供外部调用
- MCP（Model Context Protocol）协议支持
- WebSocket 实时通信

## 技术架构设计

### 轻量化架构原则
- **单进程设计**: 避免微服务复杂度
- **最小依赖**: 只引入必要第三方库
- **低资源占用**: 内存 < 50MB，CPU < 5%
- **快速启动**: 启动时间 < 2 秒

### 核心组件
1. **消息接收器**: 处理 webhook 和 API 请求
2. **规则引擎**: 条件匹配和动作生成
3. **动作执行器**: 执行具体操作
4. **配置管理**: 动态配置加载
5. **监控系统**: 运行状态监控

### 系统集成方案
```
chatshell-api (读取) → WeChat Butler (处理) → wechat-sendmsg (发送)
                    ↑                         ↓
            HTTP API/MCP 控制           chatlog-session (界面)
```

## 技术选型

### 后端技术栈
- **编程语言**: Python 3.11+（开发快速，生态丰富）
- **Web 框架**: FastAPI（轻量级，高性能）
- **LLM 接口**: litellm（统一多 provider，streaming + tool_calls 开箱即用）
- **MCP 客户端**: 官方 mcp Python SDK（StreamableHTTP 传输）
- **配置管理**: YAML + Pydantic + 环境变量
- **SSE 流式**: sse-starlette
- **日志系统**: Python logging + 结构化日志

### 前端技术栈（管理界面）
- **框架**: Vue 3 + TypeScript（与 chatlog-session 一致）
- **UI 组件**: Element Plus / Naive UI
- **状态管理**: Pinia
- **构建工具**: Vite

### 部署方案
1. **直接运行**: Python 脚本直接执行
2. **系统服务**: systemd / supervisor 托管
3. **容器化**: Docker 部署（可选）
4. **跨平台**: 支持 Windows/macOS/Linux

## 与现有系统集成

### 1. 与 chatshell-api 集成
- **方式**: MCP 协议（StreamableHTTP）+ Webhook 通知（v0.2.0）
- **配置**: MCP Server URL，空闲超时
- **功能**: 工具发现和调用（查询联系人、聊天记录等），实时消息接收（v0.2.0）

### 2. 与 wechat-sendmsg 集成
- **方式**: HTTP API 调用
- **配置**: 连接管理，错误重试
- **功能**: 消息发送，状态跟踪

### 3. 与 chatlog-session 集成
- **方式**: HTTP API + SSE 流式响应
- **配置**: API 密钥认证（X-Butler-API-Key）
- **功能**: AI 对话、模型管理、配置管理、Prompt 管理、状态监控

## 风险评估与缓解

### 技术风险
1. **微信接口稳定性**
   - 风险: wechat-sendmsg 依赖微信 GUI 自动化
   - 缓解: 定期测试适配，提供降级方案

2. **LLM 服务可用性**
   - 风险: 第三方 API 不稳定或变更
   - 缓解: 多服务商支持，本地模型备选

3. **性能瓶颈**
   - 风险: 规则数量增加导致延迟上升
   - 缓解: 规则引擎优化，缓存策略

### 项目风险
1. **开发资源不足**
   - 风险: 个人项目，开发时间有限
   - 缓解: 聚焦核心功能，渐进式开发

2. **兼容性问题**
   - 风险: 不同环境下的兼容性问题
   - 缓解: 容器化部署，兼容性测试

WeChat Butler 项目旨在填补现有微信工具链的空白，提供一个轻量级、智能化的消息自动化处理中心。通过连接 chatshell-api（读取）、wechat-sendmsg（发送）和 chatlog-session（界面），形成一个完整的微信生态解决方案。
